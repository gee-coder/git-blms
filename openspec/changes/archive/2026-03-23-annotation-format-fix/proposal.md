## Why

当前 git-blms 扩展存在两个问题：
1. **缓存失效缺陷**：git 提交后重新打开文件，仍显示旧的"未提交"标注，只有切换文件才能刷新
2. **标注格式不一致**：日期和用户名宽度不固定，导致标注栏抖动；相对时间格式过长占用过多空间

## What Changes

### 标注格式改进
- **日期格式**：绝对日期改为 `yyyy/mm/dd` 格式（4位年份，月日补零），固定10字符宽度
- **相对时间格式**：使用短格式（如 `5h ago` 代替 `5 hours ago`），固定10字符宽度，不足补空格
- **用户名显示**：限制最大6显示宽度（最多3个中文字符），超出用 `…` 截断，短名称补空格填充

### 缓存失效修复
- **问题**：git 提交不会改变文件 mtime，导致缓存键不变，旧的未提交标注被复用
- **方案**：在文件打开时检测 git HEAD 变化，或缩短缓存时间

## Capabilities

### New Capabilities
- `annotation-format`: 定义行内标注的显示格式规范，包括日期、相对时间、用户名的格式化规则
- `blame-cache-invalidation`: 定义 git blame 缓存的失效策略，确保外部 git 操作后能正确刷新

## Impact

### 受影响的代码文件
- `src/relativeTime.ts` - 日期和相对时间格式化
- `src/decoratorManager.ts` - 用户名截断和标注宽度计算
- `src/blameManager.ts` - 缓存失效逻辑
- `src/extension.ts` - 可能需要添加文件打开事件监听

### 不受影响的部分
- 颜色方案计算逻辑
- i18n 多语言支持（但需要更新翻译字符串）
- 配置项结构
